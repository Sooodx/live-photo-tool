# 产品需求文档（PRD）

## 1. 背景与目标

### 背景
使用 Panasonic 相机以 V-Log 模式拍摄的 MOV 视频，画面是低对比度的 Log 色彩，需要套入 LUT 才能还原成正常观感的 Rec.709 色彩。同时希望从视频中截取片段，导出为 Apple Live Photo 格式，方便存入 iOS 相册。

### 目标
提供一个本地 Web 工具，流程覆盖：导入视频 → 套 LUT 预览 → 裁剪片段 → 导出 Live Photo。

### 非目标
- 不做云端处理
- 不做社交分享功能
- 不支持批量处理（MVP 阶段单文件）

---

## 2. 用户场景

**典型用户**：摄影爱好者，自己用 Panasonic S 系列相机拍摄 V-Log 素材，想快速从视频中提取动态照片。

**核心操作路径**：

```
上传 MOV 文件
  → 工具生成低分辨率预览代理（proxy）
  → 用户选择 LUT（内置或自定义 .cube 文件）
  → 实时预览套 LUT 后的效果
  → 用户在时间轴上拖拽选择片段（最长 3 秒）
  → 选择封面帧（静态图）
  → 点击导出
  → 下载 ZIP 包（内含 HEIC + MOV，同名配对）
  → 导入 iOS 相册，自动识别为 Live Photo
```

---

## 3. 功能需求

### 3.1 视频导入

| 需求项 | 说明 |
|--------|------|
| 支持格式 | `.mov` / `.mp4`，H.264 / H.265 / ProRes 编码 |
| 文件大小 | 建议限制 4GB 以内（浏览器 File API 上限） |
| 上传方式 | 拖拽或点击选择，文件留在本地，仅路径传给后端 |
| 分辨率 | 主要场景为 4K（3840×2160） |

### 3.2 代理生成（Proxy）

为了浏览器预览流畅，后端在上传后自动生成低分辨率代理：

| 参数 | 值 |
|------|----|
| 代理分辨率 | 960×540（原始 4K 的 1/4） |
| 编码 | H.264，CRF 23 |
| 生成时机 | 上传完成后后台异步生成，前端轮询状态 |
| 存储位置 | 服务器临时目录，会话结束后清理 |

### 3.3 LUT 管理

**内置 LUT 列表（优先级排序）：**

| LUT 名称 | 用途 |
|----------|------|
| V-Log → V-709 | Panasonic V-Log 转 Rec.709（主力） |
| V-Log → V-709 Warm | 暖调版本 |
| V-Log → V-709 Cool | 冷调版本 |
| S-Log3 → S-709 | Sony S-Log3 转 Rec.709 |
| S-Log2 → S-709 | Sony S-Log2 转 Rec.709 |
| Log-C → Rec709 | ARRI Log-C 转 Rec.709 |

**自定义 LUT：**
- 支持用户上传 `.cube` 格式文件（33点或65点）
- 上传后出现在 LUT 列表中，本次会话有效

**LUT 强度调节：**
- 提供 0%–100% 混合滑块（blend），0% = 原始 Log，100% = 完全套用 LUT
- 默认 100%

### 3.4 预览

| 需求项 | 说明 |
|--------|------|
| 视频播放 | 基于代理文件，浏览器原生 `<video>` 播放 |
| LUT 预览 | **方案A（推荐）**：点击「预览」后端实时渲染当前帧并返回图片；**方案B**：WebGL Canvas 在前端实时应用 LUT（实现复杂但无延迟） |
| 推荐方案 | 方案A，后端用 FFmpeg 渲染单帧，延迟约 0.5–1 秒，实现简单 |
| 全屏预览 | 支持 |

### 3.5 时间轴裁剪

| 需求项 | 说明 |
|--------|------|
| 时间轴 | 显示视频缩略图帧序列 |
| 选区 | 拖拽两个手柄选择入点/出点 |
| 最大时长 | 3 秒（Apple Live Photo 规范上限） |
| 时长提示 | 实时显示已选时长，超出 3 秒时高亮警告 |
| 封面帧选择 | 在选区内拖拽选择静止封面帧（默认选区第一帧） |

### 3.6 导出

**输出文件规范（Apple Live Photo）：**

| 文件 | 规范 |
|------|------|
| `IMG_XXXX.heic` | 封面静态帧，HEIC 格式，分辨率与原视频一致（4K），嵌入 Apple 规定的 XMP 元数据 `com.apple.photos.LivePhoto.ContentIdentifier` |
| `IMG_XXXX.mov` | 视频片段，H.264 编码，AAC 音频，时长 ≤ 3 秒，嵌入相同 ContentIdentifier 到 `moov.udta` |
| ZIP 包 | 两个文件打包，文件名相同（仅扩展名不同） |

**ContentIdentifier：**
- 一个 UUID，由后端生成
- 必须同时写入 HEIC 的 XMP 和 MOV 的 `udta` box，iOS 依此配对两个文件

**导出流程：**
1. 后端对选中片段执行 FFmpeg，套 LUT，输出 MOV
2. 提取封面帧，转为 HEIC，写入 XMP
3. 对 MOV 注入 `udta` 元数据
4. 打包 ZIP，返回给前端下载

---

## 4. 非功能需求

| 项目 | 要求 |
|------|------|
| 隐私 | 所有处理在本地 Docker 容器内完成，无任何数据外传 |
| 性能 | 4K 视频 3 秒片段导出，目标 ≤ 30 秒（取决于宿主机硬件） |
| 兼容性 | 前端支持 Chrome / Safari / Firefox 最新版 |
| 会话清理 | 服务器临时文件在会话结束或 1 小时后自动删除 |

## 6. MVP 范围界定

MVP 包含：
- 单文件上传与代理生成
- 内置 V-Log→V-709 LUT + 自定义 LUT 上传
- 时间轴裁剪（最长 3 秒）
- 封面帧选择
- 导出 ZIP（HEIC + MOV）

MVP 暂不包含（后续迭代）：
- 批量处理
- LUT 强度实时预览（先做点击预览）
- 音频处理选项
- 色调曲线手动调整
